# Security Policy

The security of SkillHiive and its community is important to us.

If you believe you've discovered a security vulnerability, **please do not create a public GitHub issue.**

Instead, report it privately so we have time to investigate and fix the issue before disclosure.

---

# Reporting a Vulnerability

Please include as much information as possible:

- Description of the vulnerability
- Steps to reproduce
- Proof of concept (if available)
- Potential impact
- Suggested mitigation (optional)

Email:

security@skillhiive.com

---

# Responsible Disclosure

We ask that you:

- Give us reasonable time to investigate.
- Avoid publicly disclosing the issue before a fix is available.
- Do not exploit vulnerabilities beyond what is necessary to demonstrate the issue.
- Do not access, modify, or delete other users' data.

We are committed to responding respectfully and transparently.

---

# Supported Versions

| Version | Supported |
|----------|-----------|
| Latest Release | ✅ |
| Previous Minor Release | ✅ |
| Older Versions | ❌ |

---

# Security Philosophy

SkillHiive is built around trust.

That means:

- Privacy-first architecture
- Open source transparency
- Secure defaults
- Minimal data collection
- No unnecessary tracking

Security is an ongoing process, not a one-time feature.

Every responsible disclosure helps improve the platform for everyone.

Thank you for helping keep SkillHiive secure.
