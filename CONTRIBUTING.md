# Contributing to SkillHiive

First of all, Thank You.

Whether you're fixing a typo, reporting a bug, suggesting an idea, or contributing thousands of lines of code, you're helping make SkillHiive better.

We genuinely appreciate it.

---

# Before You Start

Please:

- Search existing Issues first.
- Read the documentation.
- Be respectful and constructive.

---

# Ways to Contribute

There are many ways to contribute.

## 🐛 Bug Reports

Open an Issue and include:

- Description
- Steps to reproduce
- Expected behaviour
- Screenshots (if applicable)
- Environment information

---

## 💡 Feature Requests

Before requesting a feature, ask yourself:

- Does this align with SkillHiive's philosophy?
- Does it encourage healthier communities?
- Does it improve collaboration?

Features that increase engagement for engagement's sake generally won't be accepted.

---

## 📖 Documentation

Good documentation is just as valuable as good code.

If something confused you, improving the docs is an excellent first contribution.

---

## 💻 Code Contributions

1. Fork the repository.

2. Create a branch.

```bash
git checkout -b feature/my-feature
```

3. Commit your changes.

```bash
git commit -m "Add my feature"
```

4. Push your branch.

```bash
git push origin feature/my-feature
```

5. Open a Pull Request.

---

# Coding Standards

Please keep code:

- Readable
- Consistent
- Well documented
- Tested where appropriate

Prefer clarity over cleverness.

---

# Pull Requests

A good Pull Request should:

- Solve one problem
- Be reasonably scoped
- Include documentation when necessary
- Explain the reasoning behind the change

---

# Philosophy

SkillHiive isn't trying to become another social platform.

Every feature should support our core values:

- Presence
- Privacy
- Intention
- Community

If a feature encourages addiction, endless scrolling, unnecessary metrics, or performance culture, it's probably not the right fit.

---

# Community

Please follow our Code of Conduct.

We want SkillHiive to be welcoming to developers of every background and experience level.

---

# Thank You

Open source succeeds because people choose to give their time, knowledge, and creativity to something bigger than themselves.

Thank you for being part of SkillHiive.
